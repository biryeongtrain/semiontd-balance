# SemionTD 밸런스 기록

실제 서버에서 사용하는 밸런스 설정과 변경 이력을 보관하는 저장소입니다.
플레이어 데이터와 서버 운영 파일은 추적하지 않습니다.

## 추적 파일

| 파일 | 내용 |
| --- | --- |
| `tower_balance.json` | 타워 능력치, 특수 능력, 업그레이드 비용 |
| `wave.json` | 라운드별 웨이브와 무한 웨이브 |
| `economy.json` | 시작 자원, 수입, 처치 보상, 타워 제한 |
| `summons.json` | 소환 몬스터 능력치와 비용 |
| `monster_scaling.json` | 시간에 따른 몬스터 성장 규칙 |
| `income_lane_routing.json` | 수입 몬스터 공격 경로 선택 규칙 |
| `leader_targeting.json` | 선두 팀 집중 공격 규칙 |
| `progression.json` | 경기 결과에 따른 진행도 보상 |
| `rating.json` | 레이팅 계산 규칙 |

## 변경 이력

변경이 확인될 때마다 한국어 커밋과 `balance-날짜-시간-커밋ID` 형식의 주석 태그를 함께 만듭니다.
태그 설명에는 설정 키가 아니라 실제 게임상 의미와 변경 전후 값이 기록됩니다.

예시:

- `돼지 타워 — 최대 체력: 300 → 340`
- `돼지 타워 — 다른 돼지 타워 수에 따른 강화 최대 중첩: 3회 → 4회`
- `염소 타워 — 반경 6블록 내 무리 타워 공격력 증가율: 10% → 12%`

## 자동 동기화

Codex 자동화는 30분마다 변경을 확인해 JSON 검증, 한국어 설명 작성, 커밋, 태그 생성과 GitHub 업로드를 수행합니다.
밸런스 값을 자동으로 수정하지 않으며, 의미를 확실하게 설명할 수 없는 변경이 있으면 업로드하지 않습니다.

구체적인 실행 규칙은 [`AUTOMATION_PROMPT.md`](AUTOMATION_PROMPT.md)에 있습니다.

## 제외 데이터

DB, 프로필, 영속성 설정, 빌드 기록, 백업본, 음악과 스카이박스 등은 `.gitignore`의 허용 목록 밖에 있으므로 Git에 포함되지 않습니다.
